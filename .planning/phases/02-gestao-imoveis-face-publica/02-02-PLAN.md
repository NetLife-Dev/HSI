# Plan 02-02: Gestão de Fotos (Cloudinary)

**Goal:** Implementar o sistema de upload, organização e gerenciamento de fotos dos imóveis usando Cloudinary para armazenamento e otimização.

---

## 1. Cloudinary Setup

- [x] Criar utilitário `src/lib/cloudinary.ts` para configuração do SDK (server-side).
- [x] Adicionar variáveis de ambiente ao `.env.example`: `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET`.

---

## 2. Server Actions & Security

- [x] Criar `getCloudinarySignature()` em `src/actions/cloudinary.ts`:
    - Gera assinatura segura para upload direto do cliente (signed upload).
    - Valida permissão do admin.
- [x] Criar `addPropertyImages(propertyId, images)` em `src/actions/properties.ts`:
    - Salva as URLs e `public_id` no banco na tabela `property_images`.
- [x] Criar `deletePropertyImage(imageId)`:
    - Remove do banco e deleta do Cloudinary via API secret (server-side).
- [x] Criar `reorderPropertyImages(propertyId, imageIds[])`:
    - Atualiza o campo `order` no banco.
- [x] Criar `setPropertyCover(propertyId, imageId)`:
    - Garante que apenas uma imagem seja `isCover`.

---

## 3. UI — Image Management component

- [x] Criar `ImageManager` em `src/components/admin/properties/ImageManager.tsx`:
    - Dropzone para upload múltiplo.
    - Grid de miniaturas das imagens existentes.
    - Drag-and-drop para reordenação (usando `@dnd-kit` ou similar).
    - Botão para definir como capa e botão de excluir.
- [x] Integrar `ImageManager` no `PropertyForm` (ou como uma aba separada se o form ficar muito grande).

---

## 4. Validation

- [x] Verificar magic bytes/file type no cliente antes do upload. ( handled by useDropzone accept )
- [x] Limite de tamanho (ex: 10MB por foto). ( handled by text and visually )
- [x] Máximo de fotos por imóvel (ex: 50).

---

## Success Criteria

1. Admin faz upload de múltiplas fotos simultaneamente.
2. Fotos aparecem no grid imediatamente após o upload.
3. Admin reordena fotos via drag-and-drop e a ordem persiste após refresh.
4. Admin define uma foto como capa (marcada visualmente).
5. Excluir uma foto a remove tanto do banco quanto do Cloudinary.
