# Plan 02-04: Página de Detalhes do Imóvel (Venda Pública)

**Goal:** Criar a página de detalhes da propriedade para o público final, exibindo fotos, descrição completa, comodidades e um formulário de "check-availability" (apenas visual por enquanto).

---

## 1. Dynamic Route & Data Fetching

- [x] Criar `/src/app/(public)/imovel/[slug]/page.tsx`:
    - Fetching de dados via Drizzle usando o `slug`.
    - Incluir `images`, `seasonalPricing` (se houver, mas focado no base por enquanto).
    - Metadata dinâmico (og:image, title, description).

---

## 2. UI Sections — Details Page

- [x] Criar **Galeria de Imagens**:
    - Grid "Estilo Airbnb" (1 grande + 4 menores) ou Carousel premium.
    - Modal/Lightbox para ver fotos em tela cheia.
- [x] Criar **Sidebar Sticky de Reserva**:
    - Seleção de datas (apenas UI).
    - Cálculo de total (limpeza + noites).
    - Botão "Reservar Agora" (direcionar para contato ou placeholder).
- [x] Criar **Conteúdo Principal**:
    - H1 (Nome), Localização.
    - Badges de Comodidades (ícones dinâmicos).
    - Descrição Rich Text (renderizar o HTML sanitizado do Tiptap).

---

## 3. Extra Features

- [x] Seção de Mapa (Google Maps ou Leaflet placeholder).
- [x] Botão de WhatsApp flutuante específico para o imóvel. (Added to sidebar)

---

## Success Criteria

1. Usuário clica em um card na Home e é levado para a página correta do imóvel.
2. Todas as fotos enviadas no admin aparecem na galeria.
3. O HTML da descrição é renderizado corretamente (negritos, listas) e sem permitir scripts maliciosos.
4. O formulário de reserva acompanha o scroll da página (sticky).
5. A página é otimizada para SEO (Title e Meta Description dinâmicos).
