## Decisions Made

1. **Schema-first all phases** — creating all tables in Phase 1 avoids "schema drift as retroactive bug"; tables used only in later phases (icalFeeds, crmLeads, financialTransactions) are created now but left empty

2. **auditLog bigserial PK** — auto-increment bigint enables range-based pagination and is better for append-only audit patterns than uuid; indexes on entityType+entityId and userId support the most common lookup patterns

3. **accounts.expiresAt as text** — the @auth/drizzle-adapter expects this column as text; using integer would cause silent auth failures at runtime (documented in plan research)

4. **Startup migration via instrumentation.ts** — not at build time, so EasyPanel redeploys are fully automated; no `npx drizzle-kit migrate` shell step required in nixpacks.toml

5. **src/lib/audit.ts stub** — Plan 01-03 imports `logAction` from this path; stub prevents build-time module-not-found error; Plan 01-05 replaces with real DB-writing implementation
